# 方剂数据查证方法论（确保出自经典）

> 目标：对 SQLite 数据库 `db/zhongyi.db` 的 `formulas` 表（及迁移前 `assets/data/formulas.json`）中的 **每一个方剂**，逐条核查其全部字段是否出自公认中医经典，
> 并将查证结果记录在数据本身的 `verification` 字段中，同时在本仓库 `docs/formulas_verification.md`
> 生成逐条可读报告。

## 一、查证的权威依据（经典基准清单）

以历代公认的方剂学、临床经典为基准，优先级从高到低：

| 层级 | 典籍 | 时期/作者 |
|------|------|-----------|
| 1 | 《伤寒论》《金匮要略》 | 汉·张仲景 |
| 2 | 《备急千金要方》《千金翼方》 | 唐·孙思邈 |
| 3 | 《外台秘要》 | 唐·王焘 |
| 4 | 《太平惠民和剂局方》 | 宋·太医局 |
| 5 | 《脾胃论》《兰室秘藏》 | 金元·李东垣 |
| 6 | 《丹溪心法》 | 元·朱丹溪 |
| 7 | 《景岳全书》 | 明·张景岳 |
| 8 | 《医方集解》 | 清·汪昂 |
| 9 | 《温病条辨》 | 清·吴鞠通 |
| 10 | 《医林改错》 | 清·王清任 |
| 11 | 《医宗金鉴》 | 清·吴谦 等 |
| 12 | 《太平圣惠方》《圣济总录》 | 宋·官修 |
| 13 | 其他公认经典方书（如《宣明论方》《医学心悟》《辨证录》等） |

> 现代教材（五版/十版《方剂学》《中药学》）仅作为"通行记载"参考，最终以原典为据。

## 二、核对的维度（每个方剂逐项检查）

对每个方剂逐一核查以下字段，任一不符即记为问题：

1. **方名（name）**：方名是否真实存在于上述经典；是否存在杜撰、张冠李戴。
2. **药物组成（ingredients）**：每味药的名称、用量、君臣佐使角色是否与经典一致；
   是否存在经典未载的药物、错误的药量、错误的配伍。
3. **功效（effects）/ 主治（indications）**：是否与经典方义、后世通行方解相符。
4. **剂型与命名一致性（dosage form）**：方名后缀（汤/丸/散/膏/丹/饮/煎…）与实际剂型是否相符。
   - 注意：中医历史上散剂、丸剂后世多"改作汤剂水煎服"（如银翘散、逍遥散今多作汤服），
     这类"名散实汤"属 **历史演变**，不算错误，但需在 notes 标注原剂型与今用剂型。
   - 真正的"不符"指：方名所暗示的剂型与任何经典记载都矛盾，或来源标注自相矛盾。
5. **来源典籍（source）/ 作者（author）**：标注的出处是否真实、是否张冠李戴。
6. **用法（usage）**：是否与所标剂型的制备方法一致（含历史演变说明）。

## 三、查证方法（如何"确保"出自经典）

1. **AI 知识库初核**：基于训练语料中收录的上述经典原文与通行方解，逐一判断各维度。
2. **必要 web 检索/原文比对**：对 AI 不确定、存疑或来源可疑者，使用网络检索比对原典条文。
3. **存疑即标记**：凡 AI 无法在知识范围内确认的，一律标记 `needs_review`，交由人工查阅原典，
   **不臆断为正确**。
4. **逐条记录**：每查一个方剂，立即写入其 `verification` 字段并追加到报告 md，做到"查一个记一个"，
   不批量下结论。

## 四、数据中的 verification 字段 schema

`formulas` 表（或原 `formulas.json`）每个方剂记录新增：

```json
"verification": {
  "status": "verified_classic | mismatch | needs_review",
  "classic_source": "《伤寒论》",
  "checked_by": "AI知识库核对",
  "checked_at": "2026-07-17",
  "name_ok": true,
  "composition_ok": true,
  "dosage_form_ok": true,
  "source_ok": true,
  "notes": "名称、组成、来源均见于《伤寒论》原文；剂型'汤'与用法'水煎服'一致。",
  "issues": []
}
```

状态定义：

- `verified_classic`：名称、组成、来源、剂型命名均与公认经典一致（含已说明的历史演变）。
- `mismatch`：存在明确不符（组成错误、来源张冠李戴、或剂型命名与原典矛盾且无法解释为历史演变）。
- `needs_review`：AI 无法在知识范围内确认/存疑，需人工查阅原典复核。

`issues` 数组记录具体问题描述，例如：
`["方名'XX散'原典为散剂，但 source 标注《温病条辨》且该方实际出自《太平惠民和剂局方》，出处张冠李戴"]`。

## 五、局限声明（重要）

- AI 查证以训练语料中的经典记载为据，**不等同于逐字核校原典**。
- 凡标记 `verified_classic` 者应视为"高可信度、建议人工抽检"，`needs_review` 必须人工复核后方可采用。
- 本查证不改变方剂临床内容，仅补充出处可信度标注；发现明确错误时另立 issue 清单供修订。
