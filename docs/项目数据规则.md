# 中医辨证论治系统 - 项目数据规则

> 最后更新：2026-07-14

---

## 一、项目概述

本项目是一个中医辨证论治知识应用，包含证型、方剂、中药、穴位、针方、经络、治法、功效、中西对照等 10 类数据，总计 **1705 条**记录。支持列表浏览、详情查看、知识图谱、全文搜索、收藏等功能。

## 三、ID 命名规范

- 格式：`{type}_{序号}`，序号为三位数字
- 示例：`syndrome_001`、`formula_012`、`medicine_504`
- 经络特殊 ID：`extra_point`（经外奇穴）
- 所有关联字段均使用完整 ID 字符串引用

## 四、数据关联规则

### 4.1 核心关联图

```
证型 ──→ 方剂（syndrome.related_formulas / formula.syndrome_ids）
证型 ──→ 针方（syndrome.related_needle）
证型 ──→ 治法（treatment.related_syndromes）
证型 ──→ 中西对照（mapping.related_syndrome）
方剂 ──→ 中药（formula.ingredients[].medicine_id）
方剂 ──→ 证型（formula.syndrome_ids）
针方 ──→ 穴位（needle.acupoints[].acupoint_id）
针方 ──→ 证型（needle.related_syndromes）
穴位 ──→ 经络（acupoint.meridian_id）
中药 ──→ 功效（medicine.effects）
经络 ──→ 穴位（meridian.related_acupoints）
```

### 4.2 关联完整性要求

- **证型必须**关联方剂、针方、治法、中西对照（四项全覆盖）
- **方剂必须**关联证型和中药
- **针方必须**关联穴位和证型
- **穴位必须**关联经络
- **中西对照**必须关联一个证型

## 十三、数据添加规范

### 13.1 新增证型
1. ID 格式：`syndrome_{下一序号}`（三位数字）
2. 必填字段：`id`、`name`、`pinyin`、`classification`、`diagnosis_points`
3. 必须同时更新：`formulas.json`（添加 syndrome_ids）、`treatments.json`（添加 related_syndromes）、`modern_mapping.json`（添加 related_syndrome）

### 13.2 新增方剂
1. ID 格式：`formula_{下一序号}`
2. 必填字段：`id`、`name`、`pinyin`、`category`、`ingredients`（含 medicine_id 引用）
3. 关联要求：至少关联 1 个证型（syndrome_ids）、包含药材引用

### 13.3 新增中西对照
1. ID 格式：`mapping_{下一序号}`
2. 必填字段：`id`、`chinese_term`、`modern_term`、`category`、`comparison`（含 aspect/tcm/western）
3. 必须关联 `related_syndrome`

### 13.4 通用规则
- 所有 ID 必须全局唯一
- 所有关联引用必须指向存在的实体
- JSON 文件使用 UTF-8 编码、2 空格缩进
