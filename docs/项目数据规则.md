# 中医辨证论治系统 - 项目数据规则

> 最后更新：2026-07-17

---

## 一、项目概述

本项目是一个中医辨证论治知识应用，包含证型、方剂、中药、穴位、针方、经络、治法、功效、中西对照等 9 类数据，总计 **3179 条**记录。数据统一存放于 SQLite 数据库 `db/zhongyi.db`，支持列表浏览、详情查看、知识图谱、全文搜索、收藏等功能。

---

---

## 三、ID 命名规范

- 格式：`{type}_{序号}`，序号为三位数字
- 示例：`syndrome_001`、`formula_012`、`medicine_504`
- 经络特殊 ID：`extra_point`（经外奇穴）
- 所有关联字段均使用完整 ID 字符串引用

---

## 四、数据关联规则

### 4.1 核心关联图

```
证型 ──→ 方剂（syndrome.related_formulas / formula.syndrome_ids）
证型 ──→ 针方（syndrome.related_needle）
证型 ──→ 治法（treatment.related_syndromes）
证型 ──→ 中西对照（mapping.related_syndrome）
方剂 ──→ 中药（formula.ingredients[].medicine_id）
方剂 ──→ 证型（formula.syndrome_ids）
针方 ──→ 穴位（needle.acupoints[].acupoint_id）
针方 ──→ 证型（needle.related_syndromes）
穴位 ──→ 经络（acupoint.meridian_id）
中药 ──→ 功效（medicine.effects）
经络 ──→ 穴位（meridian.related_acupoints）
```

### 4.2 关联完整性要求

- **证型必须**关联方剂、针方、治法、中西对照（四项全覆盖）
- **方剂必须**关联证型和中药
- **针方必须**关联穴位和证型
- **穴位必须**关联经络
- **中西对照**必须关联一个证型

## 十三、数据维护规范

数据现已统一存储于 SQLite 数据库，编辑应通过 SQL 或 `scripts/` 下的 Python 工具完成，不再直接手写 JSON。

### 13.1 新增证型
1. ID 格式：`syndrome_{下一序号}`（三位数字），写入 `syndromes` 表
2. 必填字段：`id`、`name`、`pinyin`、`classification`（写入 `syndromes_classification` 子表）、`diagnosis_points`
3. 关联：通过 `syndromes_related_formulas` / `syndromes_related_needle` / `syndromes_related_treatments` / `syndromes_related_effects` 子表建立，并执行双向闭包（见 13.4）

### 13.2 新增方剂
1. ID 格式：`formula_{下一序号}`，写入 `formulas` 表
2. 必填字段：`id`、`name`、`pinyin`、`category`；组成写入 `formulas_ingredients` 子表（含 `medicine_id` 引用）
3. 关联：至少关联 1 个证型（`formulas_syndrome_ids`），并通过 `formulas_related_syndromes` 与对方闭包

### 13.3 新增中西对照
1. ID 格式：`mapping_{下一序号}`，写入 `modern_mapping` 表
2. 必填字段：`id`、`chinese_term`、`modern_term`、`category`、`comparison`（含 aspect/tcm/western）
3. 关联：通过 `related_syndrome` 指向证型

### 13.4 通用规则
- 所有 ID 必须全局唯一
- 所有关联引用必须指向存在的实体（无悬空引用）
- 双向关联须对称：新增正向引用后，须同步回填反向引用（可由 `scripts/clean_data.py` 的闭包逻辑完成）
- 修改 `db/zhongyi.db` 后，须同步 `public/zhongyi.db` 并重新 `npm run build`

## 十四、数据验证

数据质量与一致性检查统一通过 `scripts/` 下的 Python 工具（只读分析 + 安全清洗）：

```bash
# 1. 全程引用完整性 / 一致性 / 覆盖率核查（不修改数据）
python3 scripts/check_sqlite.py db/zhongyi.db

# 2. 字段填充率、双向缺口、重名、格式问题等质量分析
python3 scripts/analyze_data.py db/zhongyi.db

# 3. 安全清洗（去重、回填、双向闭包补全；接受 --db 指定库路径）
python3 scripts/clean_data.py --db db/zhongyi.db

# 4. 将 SQLite 反向导出为 JSON 备份（供审阅/留档）
python3 scripts/export_json_backup.py db/zhongyi.db
```

校验目标：悬空引用 = 0、双向不对称 = 0、ID 唯一、重名 = 0、字段填充率达标。修改后务必同步 `public/zhongyi.db` 并重新构建。
